# UI Rules (must-follow)

1) Never invent styles per-page. Use tokens and base components (Button/Card/Container/AppHeader).
2) Background must follow the gradient base defined in globals.css.
3) Glass UI uses the `.glass` class or Card component (no random opacity values).
4) Theme must be controlled only via ThemeToggle (system/light/dark).
5) Hero pages (Landing/Auth) use the same typography scale and spacing.
